## 摘要
服务系统的性能取决于参与者对设计选择的反应，但由于人类行为的复杂性，对这些反应进行建模非常困难。我们提出了一种大语言模型驱动的多智能体模拟（LLM-MAS）框架，用于优化服务运营。我们将该问题建模为具有决策依赖型不确定性的随机优化问题：设计选择被嵌入到提示词（prompts）中，并塑造了由大语言模型驱动的智能体在交互过程中产生的结果分布。通过在提示词中嵌入关键数值信息并从大语言模型生成的文本中提取这些信息，我们将这种不确定性建模为一个受控马尔可夫链（controlled Markov chain）。我们开发了一种在线轨迹学习算法，在单次模拟运行中，能够同时构建零阶梯度估计并更新设计参数，以优化稳态性能。我们还结合了方差缩减技术。在可持续供应链的应用中，我们的方法优于基准方法，包括黑盒优化以及将大语言模型用作数值求解器或角色扮演系统设计者的方法。一项基于真实行为数据的最优竞赛设计案例研究表明，LLM-MAS 既是已知设计方案的高性价比评估器，也是一种能够发现传统方法所忽略的优秀设计的探索性工具。

## Abstract
Service system performance depends on how participants respond to design choices, but modeling these responses is hard due to the complexity of human behavior. We introduce an LLM-powered multi-agent simulation (LLM-MAS) framework for optimizing service operations. We pose the problem as stochastic optimization with decision-dependent uncertainty: design choices are embedded in prompts and shape the distribution of outcomes from interacting LLM-powered agents. By embedding key numerical information in prompts and extracting it from LLM-generated text, we model this uncertainty as a controlled Markov chain. We develop an on-trajectory learning algorithm that, on a single simulation run, simultaneously constructs zeroth-order gradient estimates and updates design parameters to optimize steady-state performance. We also incorporate variance reduction techniques. In a sustainable supply chain application, our method outperforms benchmarks, including blackbox optimization and using LLMs as numerical solvers or as role-playing system designers. A case study on optimal contest design with real behavioral data shows that LLM-MAS is both as a cost-effective evaluator of known designs and an exploratory tool that can uncover strong designs overlooked by traditional approaches.

*   **核心思想**：将设计参数 $\theta$ 嵌入到 LLM 的提示词（Prompts）中，通过 LLM 生成的文本响应来模拟智能体的决策。系统状态的演变被建模为一个**受控马尔可夫链（Controlled Markov Chain）**。

    *   **语义接口**：通过提示词模板将数值型设计参数转化为文本，再从 LLM 生成的 JSON 格式文本中提取数值型动作。
    *   **在线轨迹学习（On-Trajectory Learning, OTL）算法**：
        *   **零阶梯度估计**：由于 LLM 的输出分布不可导且复杂，采用基于随机扰动的零阶方法估计梯度。
        *   **单轨迹更新**：为了节省昂贵的 LLM 调用成本，算法在单次模拟轨迹中同时进行梯度估计和参数更新，而非多次运行模拟。
        *   **双时间尺度方案**：梯度估计在快尺度上演化，设计参数在慢尺度上更新，以确保收敛性。
    *   **方差缩减技术**：引入了“引导扰动（Guided Perturbation）”和“残差反馈（Residual Feedback）”来应对多智能体交互带来的高噪声。

### 6. 主要结论与发现
*   **算法优越性**：提出的 OTL 算法及其方差缩减变体在优化效率和最终效果上显著优于贝叶斯优化和直接提示 LLM 的方法。
*   **LLM 的角色**：LLM 是传统运筹学（OR）方法的**补充而非替代品**。单纯依靠 LLM 的直觉设计或数值求解效果不佳，必须结合严谨的随机优化框架。
*   **行为保真度**：LLM-MAS 能够有效复制人类在复杂激励机制下的异质性行为，发现传统解析模型可能忽略的最优设计（例如，发现降低入场费反而能通过增加参与度来提升总努力值）。

### 7. 优点
*   **弥合理论与现实**：通过 LLM 捕捉难以量化的心理和行为因素，使优化结果更具鲁棒性。
*   **高效性**：OTL 算法极大地减少了模拟所需的 Token 消耗和时间成本。
*   **可解释性**：LLM 生成的“推理轨迹（Reasoning Traces）”为设计者提供了智能体决策背后的逻辑，这是传统强化学习或黑盒模型无法提供的。

### 8. 不足与局限
*   **维度灾难**：零阶优化方法在处理极高维度的设计空间时，效率会显著下降。
*   **LLM 固有偏差**：模拟的准确性受限于底层 LLM 的预训练分布，可能存在文化或特定场景下的行为偏差。
*   **查询成本**：尽管 OTL 进行了优化，但对于超大规模、多智能体的实时系统，LLM 的推理成本和延迟仍是实际应用的瓶颈。
*   **应用限制**：目前主要集中在稳态性能优化，对于极短时间内的动态博弈捕捉可能尚不充分。
