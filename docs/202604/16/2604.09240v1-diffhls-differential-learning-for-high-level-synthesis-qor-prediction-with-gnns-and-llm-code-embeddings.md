## 摘要
高级综合（HLS）将 C/C++ 编译为 RTL，但由于每个设计点都需要耗时的综合过程，探索基于 pragma 驱动的优化选择仍然代价高昂。我们提出了 DiffHLS，这是一个用于 HLS 结果质量（QoR）预测的差分学习框架，它通过内核-设计对（即内核基准和插入 pragma 的设计变体）进行学习。DiffHLS 使用专门的图神经网络（GNN）分支对内核和设计的中间表示（IR）图进行编码，并利用预训练代码大语言模型（LLM）的代码嵌入来增强增量（delta）路径。与直接回归绝对目标不同，我们联合预测内核基准和设计引起的增量，并将其组合以获得最终的设计预测。在 PolyBench 数据集上，DiffHLS 在四种 GNN 骨干网络下的平均 MAPE 均低于 GNN 基准模型，且 LLM 代码嵌入相比仅使用 GNN 的消融实验展现出持续的改进。我们进一步在 ForgeHLS 数据集上验证了该框架的可扩展性。

## Abstract
High-Level Synthesis (HLS) compiles C/C++ into RTL, but exploring pragma-driven optimization choices remains expensive because each design point requires time-consuming synthesis. We propose \textbf{\DiffHLS}, a differential learning framework for HLS Quality-of-Result (QoR) prediction that learns from kernel--design pairs: a kernel baseline and a pragma-inserted design variant. \DiffHLS~encodes kernel and design intermediate-representation graphs with dedicated graph neural network (GNN) branches, and augments the delta pathway with code embeddings from a pretrained code large language model (LLM). Instead of regressing absolute targets directly, we jointly predict the kernel baseline and the design-induced delta, and compose them to obtain the design prediction. On PolyBench, \DiffHLS~attains lower average MAPE than GNN baselines under four GNN backbones, and LLM code embeddings consistently improve over a GNN-only ablation. We further validate scalability on the ForgeHLS dataset.

*   **核心问题**：在高层次综合（HLS）中，通过插入 Pragma 指令（如流水线、展开等）优化设计空间非常耗时，因为每个设计点都需要经过漫长的综合过程。现有的基于图神经网络（GNN）的预测方法在处理不同内核（Kernel）间巨大的目标方差，以及捕捉源代码中显式的 Pragma 语义信息方面存在不足。
*   **研究动机**：
    1.  **预测变化比预测绝对值更精确**：同一内核的不同设计变体共享大部分基础结构，预测 Pragma 带来的“增量”能降低学习难度。
    2.  **图与代码信息互补**：中间表示（IR）图捕捉结构依赖，而源代码保留了 Pragma 指令和局部语义，结合两者可提升预测精度。

### 2. 方法论
*   **核心思想**：提出**差分学习（Differential Learning）**框架。将设计目标 $y_d$ 分解为内核基准值 $y_k$ 和设计诱导的增量 $\Delta$（即 $y_d = y_k + \Delta$）。
*   **关键技术细节**：
    *   **双分支图编码**：使用两个参数独立的 GNN 分支分别对内核 IR 图和设计 IR 图（CDFG）进行编码，提取结构特征。
    *   **LLM 代码嵌入**：引入冻结的预训练代码大语言模型（如 Qwen2.5-Coder），提取设计源代码的隐藏层状态作为语义特征，通过适配器（Adapter）注入增量预测路径。
    *   **差分预测头**：
        *   **内核头**：仅根据内核图特征预测基准值 $y_k$。
        *   **增量头**：结合内核特征、设计特征和 LLM 嵌入预测 $\Delta$。
    *   **联合损失函数**：使用 SmoothL1 损失同时监督 $y_k$、$\Delta$ 以及组合后的 $y_d$，确保预测的一致性并防止误差累积。

*   **架构创新**：通过差分学习巧妙地简化了回归问题的复杂度，将“大方差”问题转化为“小增量”预测。
*   **多模态融合**：结合了编译器 IR 的结构化信息和 LLM 的语义理解能力，充分利用了现有预训练模型的知识。
*   **诊断性强**：双头设计允许开发者分析误差是来源于基准估计还是增量建模。

### 8. 不足与局限
*   **LLM 开销**：虽然 LLM 是冻结的，但提取嵌入仍需一定的计算资源，且模型体积较大，可能限制其在轻量级工具链中的集成。
