## 摘要
服务系统的性能取决于参与者对设计选择的反应，但由于人类行为的复杂性，对这些反应进行建模具有挑战性。我们引入了一个大语言模型驱动的多智能体模拟（LLM-MAS）框架，用于优化服务运营。我们将该问题表述为具有决策依赖不确定性的随机优化：设计选择被嵌入到提示词中，并塑造了相互作用的 LLM 驱动智能体产生结果的分布。通过在提示词中嵌入关键数值信息并从 LLM 生成的文本中提取信息，我们将这种不确定性建模为受控马尔可夫链。我们开发了一种在线轨迹学习算法，在单次模拟运行中，同时构建零阶梯度估计并更新设计参数，以优化稳态性能。我们还结合了方差缩减技术。在可持续供应链的应用中，我们的方法优于基准方法，包括黑盒优化以及将 LLM 用作数值求解器或角色扮演系统设计者。一项结合真实行为数据的最优竞赛设计案例研究表明，LLM-MAS 既是已知设计的成本效益型评估器，也是一种能够发现传统方法所忽视的优异设计的探索性工具。

## Abstract
Service system performance depends on how participants respond to design choices, but modeling these responses is hard due to the complexity of human behavior. We introduce an LLM-powered multi-agent simulation (LLM-MAS) framework for optimizing service operations. We pose the problem as stochastic optimization with decision-dependent uncertainty: design choices are embedded in prompts and shape the distribution of outcomes from interacting LLM-powered agents. By embedding key numerical information in prompts and extracting it from LLM-generated text, we model this uncertainty as a controlled Markov chain. We develop an on-trajectory learning algorithm that, on a single simulation run, simultaneously constructs zeroth-order gradient estimates and updates design parameters to optimize steady-state performance. We also incorporate variance reduction techniques. In a sustainable supply chain application, our method outperforms benchmarks, including blackbox optimization and using LLMs as numerical solvers or as role-playing system designers. A case study on optimal contest design with real behavioral data shows that LLM-MAS is both as a cost-effective evaluator of known designs and an exploratory tool that can uncover strong designs overlooked by traditional approaches.

### 2. 方法论：LLM-MAS 框架与在线学习算法
*   **核心思想**：将系统设计优化建模为具有“决策依赖不确定性”的随机优化问题。设计参数 $\theta$ 被嵌入到智能体的提示词（Prompts）中，通过 LLM 的自回归生成产生行为，进而影响系统状态的演化。
*   **关键技术细节**：
    *   **受控马尔可夫链（Controlled Markov Chain）**：将 LLM 智能体的观察、沟通和行动提取为数值变量，构建马尔可夫状态转移模型。
    *   **在线轨迹学习（On-Trajectory Learning, OTL）**：为了解决 LLM 推理延迟高、成本贵的问题，算法在单次模拟轨迹中同时进行梯度估计和参数更新，无需多次独立运行。
    *   **零阶梯度估计（Zeroth-Order Optimization）**：由于 LLM 的生成过程不可微且分布复杂，采用基于随机扰动的零阶方法估计梯度。
    *   **方差缩减技术**：引入“引导扰动（Guided Perturbation）”和“残差反馈（Residual Feedback）”来降低模拟过程中的噪声，提高收敛效率。
*   **算法流程**：在每一轮模拟中，对当前设计参数进行微小扰动，观察 LLM 智能体的反馈，计算性能差异，并利用双时间尺度（Two-timescale）方案更新设计参数。

### 6. 主要结论与发现
*   **算法优越性**：提出的 OTL 算法在收敛速度和最终性能上显著优于贝叶斯优化和纯 LLM 启发式方法。
*   **LLM 模拟的有效性**：LLM 能够准确复现人类在竞赛中的非理性行为（如风险厌恶、损失厌恶），而这些行为是传统博弈论模型无法预测的。
*   **发现新设计**：在竞赛设计中，LLM-MAS 发现了一种“中等入场费、低保留价”的方案，其产生的总努力值远高于博弈论预测的最优解。
*   **互补性**：LLM 并非传统运筹学（OR）方法的替代品，而是作为模拟器与严谨的数值优化算法相结合，才能发挥最大效用。

### 7. 优点与亮点
*   **端到端处理非结构化信息**：LLM 可以直接处理广告文案、评论等文本信息对决策的影响，这是传统数值模拟器难以做到的。
*   **高效率采样**：OTL 算法极大地减少了所需的 Token 数量，使 LLM 驱动的优化在经济上具有可行性。
*   **可解释性**：通过提取 LLM 的推理链（Chain-of-Thought），设计者可以理解智能体为何对特定政策做出反应，而不仅仅是得到一个数值结果。

### 8. 不足与局限
*   **维度灾难**：零阶优化方法在处理超高维设计空间（如成百上千个参数）时效率会下降。
*   **模型偏差风险**：模拟的准确性高度依赖于底层 LLM 的质量，如果模型存在固有偏见，优化结果可能会误导现实决策。
*   **成本与延迟**：尽管 OTL 优化了效率，但相比纯数值模拟，LLM-MAS 的运行成本和时间成本依然较高。
*   **应用限制**：目前主要集中在稳态性能优化，对于极短期的动态反应模拟可能仍有挑战。
