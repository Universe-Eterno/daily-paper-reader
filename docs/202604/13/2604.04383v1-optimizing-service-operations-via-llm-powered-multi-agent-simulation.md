## 摘要
服务系统的性能取决于参与者对设计选择的反应，但由于人类行为的复杂性，对这些反应进行建模非常困难。我们提出了一种大语言模型驱动的多智能体模拟（LLM-MAS）框架，用于优化服务运营。我们将该问题建模为具有决策依赖型不确定性的随机优化问题：设计选择被嵌入到提示词（prompts）中，并塑造了由大语言模型驱动的相互作用智能体所产生的结果分布。通过在提示词中嵌入关键数值信息并从大语言模型生成的文本中提取这些信息，我们将这种不确定性建模为受控马尔可夫链（controlled Markov chain）。我们开发了一种在线轨迹学习算法，在单次模拟运行中，能够同时构建零阶梯度估计并更新设计参数，以优化稳态性能。我们还引入了方差缩减技术。在可持续供应链的应用中，我们的方法优于基准方法，包括黑盒优化以及将大语言模型用作数值求解器或角色扮演系统设计者的方法。一项基于真实行为数据的最优竞赛设计案例研究表明，LLM-MAS 既是已知设计的成本效益型评估器，也是一种能够发现传统方法所忽略的优秀设计的探索性工具。

## Abstract
Service system performance depends on how participants respond to design choices, but modeling these responses is hard due to the complexity of human behavior. We introduce an LLM-powered multi-agent simulation (LLM-MAS) framework for optimizing service operations. We pose the problem as stochastic optimization with decision-dependent uncertainty: design choices are embedded in prompts and shape the distribution of outcomes from interacting LLM-powered agents. By embedding key numerical information in prompts and extracting it from LLM-generated text, we model this uncertainty as a controlled Markov chain. We develop an on-trajectory learning algorithm that, on a single simulation run, simultaneously constructs zeroth-order gradient estimates and updates design parameters to optimize steady-state performance. We also incorporate variance reduction techniques. In a sustainable supply chain application, our method outperforms benchmarks, including blackbox optimization and using LLMs as numerical solvers or as role-playing system designers. A case study on optimal contest design with real behavioral data shows that LLM-MAS is both as a cost-effective evaluator of known designs and an exploratory tool that can uncover strong designs overlooked by traditional approaches.

### 2. 方法论
*   **核心思想**：将系统设计建模为具有**决策依赖型不确定性（Decision-Dependent Uncertainty）**的随机优化问题。设计参数 $\theta$ 被嵌入到智能体的提示词（Prompts）中，通过影响 LLM 生成 Token 的概率分布，进而改变智能体的行为和系统最终产出的分布。
*   **关键技术细节**：
    *   **受控马尔可夫链（Controlled Markov Chain）**：将 LLM 智能体的观察、沟通、行动和记忆定义为系统状态 $\xi$，通过提示词模板实现数值与文本的端到端转换，将模拟过程形式化为马尔可夫过程。
    *   **在线轨迹学习算法（On-Trajectory Learning, OTL）**：
        *   **零阶梯度估计**：由于 LLM 生成过程的梯度不可导且极其复杂，采用基于随机扰动的零阶方法估计梯度。

        *   **双时间尺度（Two-timescale）**：梯度估计在快尺度上演化，设计参数在慢尺度上更新，以确保收敛性。
    *   **方差缩减技术**：引入了**引导扰动（Guided Perturbation）**和**残差反馈（Residual Feedback）**，利用已知函数结构或历史迭代信息来降低零阶估计的高方差。

### 6. 主要结论与发现
*   **行为一致性**：LLM 智能体能够很好地复制人类在竞赛中的行为（如风险规避、损失厌恶），其表现与实验室数据高度吻合，而显著偏离传统的博弈论预测。
*   **算法优越性**：OTL 算法在相同 LLM 查询预算下，比黑盒优化和直接使用 LLM 决策更有效。
*   **设计发现**：在竞赛设计中，LLM-MAS 发现了一种“中等入场费、低保留价”的设计方案，其产生的总努力程度远高于博弈论推导的最优解。
*   **互补性**：LLM 并非传统运筹学（OR）方法的替代品，而是补充。严谨的随机优化框架对于发挥 LLM 的模拟潜力至关重要。

### 7. 优点
*   **桥接理论与现实**：通过 LLM 捕捉难以量化的心理和行为因素，缩小了优雅理论模型与复杂现实系统之间的差距。
*   **高效率模拟**：提出的 OTL 算法显著降低了利用 LLM 进行模拟优化的成本。
*   **可解释性**：LLM 生成的推理轨迹（Reasoning Traces）为设计者提供了关于“为什么某种设计有效/无效”的定性洞察。

### 8. 不足与局限
*   **高维瓶颈**：零阶优化方法在处理极高维度的设计空间时，收敛速度会显著变慢。
*   **保真度风险**：虽然在特定案例中表现良好，但 LLM 智能体是否在所有服务场景下都能准确模拟人类行为仍需进一步验证（存在幻觉或训练偏差风险）。
*   **成本问题**：尽管 OTL 减少了查询，但对于超大规模智能体系统（如数万名代理），Token 成本和推理延迟依然是巨大的挑战。
