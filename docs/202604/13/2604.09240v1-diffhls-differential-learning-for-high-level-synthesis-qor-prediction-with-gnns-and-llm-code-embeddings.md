## 摘要
高级综合 (HLS) 将 C/C++ 编译为 RTL，但由于每个设计点都需要耗时的综合过程，探索基于 pragma 的优化选择仍然代价高昂。我们提出了 DiffHLS，这是一个用于 HLS 结果质量 (QoR) 预测的差分学习框架，它从“内核-设计”对（即内核基准和插入 pragma 的设计变体）中进行学习。DiffHLS 使用专门的图神经网络 (GNN) 分支对内核和设计的中间表示 (IR) 图进行编码，并利用预训练代码大语言模型 (LLM) 的代码嵌入来增强增量 (delta) 路径。我们并非直接回归绝对目标，而是联合预测内核基准和设计引起的增量，并将它们组合以获得最终的设计预测。在 PolyBench 数据集上，DiffHLS 在四种 GNN 骨干网络下的平均绝对百分比误差 (MAPE) 均低于 GNN 基准模型，且 LLM 代码嵌入相比仅使用 GNN 的消融实验表现出持续的改进。我们在 ForgeHLS 数据集上进一步验证了其可扩展性。

## Abstract
High-Level Synthesis (HLS) compiles C/C++ into RTL, but exploring pragma-driven optimization choices remains expensive because each design point requires time-consuming synthesis. We propose \textbf{\DiffHLS}, a differential learning framework for HLS Quality-of-Result (QoR) prediction that learns from kernel--design pairs: a kernel baseline and a pragma-inserted design variant. \DiffHLS~encodes kernel and design intermediate-representation graphs with dedicated graph neural network (GNN) branches, and augments the delta pathway with code embeddings from a pretrained code large language model (LLM). Instead of regressing absolute targets directly, we jointly predict the kernel baseline and the design-induced delta, and compose them to obtain the design prediction. On PolyBench, \DiffHLS~attains lower average MAPE than GNN baselines under four GNN backbones, and LLM code embeddings consistently improve over a GNN-only ablation. We further validate scalability on the ForgeHLS dataset.

#### 1. 核心问题与整体含义（研究动机和背景）
在高级综合（HLS）流程中，设计人员通过在 C/C++ 代码中插入 Pragma 指令（如流水线化、循环展开等）来优化硬件性能。然而，评估这些优化后的结果质量（QoR，如资源占用和时序）通常需要运行完整的 HLS 综合工具，这极其耗时。现有的基于图神经网络（GNN）的预测方法面临两个主要挑战：
*   **跨内核方差大**：不同算法内核之间的 QoR 绝对值差异巨大，模型难以同时捕捉内核间的巨大差异和内核内 Pragma 带来的细微变化。
*   **语义信息丢失**：编译器中间表示（IR）图在降低抽象层次时，可能会弱化源代码中显式的 Pragma 指令信息。

#### 2. 方法论：核心思想与技术细节
DiffHLS 提出了一种**差分学习（Differential Learning）**框架，其核心思想是将设计点的预测分解为“内核基准值”和“Pragma 引起的增量”：
*   **差分公式**：$y_{design} = y_{kernel} + \Delta$。模型不再直接回归绝对值，而是学习内核基准 $y_k$ 和增量 $\Delta$。
*   **双分支图编码**：使用两个参数独立的 GNN 分支分别处理原始内核和设计变体的控制数据流图（CDFG）。
*   **LLM 代码嵌入融合**：引入预训练的代码大语言模型（如 Qwen2.5-Coder-1.5B），提取设计源代码的隐藏层表示（Frozen Embedding）。这些嵌入作为辅助信号注入增量预测路径，以捕捉 Pragma 的语义。
*   **联合训练目标**：损失函数包含三部分：内核基准预测误差、增量预测误差、以及最终组合结果与真实值的误差（一致性约束），均采用 SmoothL1 损失。

#### 6. 主要结论与发现

*   **架构创新**：将差分学习引入 HLS 预测，巧妙解决了跨内核数据分布不均的问题。
*   **多模态融合**：结合了 IR 图的结构信息和源代码的语义信息，弥补了单一表示的不足。
*   **即插即用**：该框架不依赖特定的 GNN 算子，具有很强的通用性和鲁棒性。

#### 8. 不足与局限
*   **LLM 开销**：虽然 LLM 是冻结的，但提取嵌入仍需一定的计算资源，且模型体积较大，可能不适合极轻量级的 EDA 工具。
*   **输入长度限制**：LLM 对代码 Token 长度有限制，对于超大规模的 C++ 源文件，可能存在截断风险或信息丢失。
*   **依赖原始内核**：该方法要求已知原始内核（无 Pragma 版本）的综合结果作为基准，这在某些完全从零开始的设计探索场景中可能受限。
