## 摘要
在许多现实场景中，需要解决的问题实例非常相似，以往优化运行中的知识可以被潜在地利用。我们针对带时间窗的旅行商问题（TSPTW）探讨了这一点，该问题通常出现在旅行时间矩阵固定但时间窗约束在相关任务之间发生变化的情境中。然而，现有的 TSPTW 研究尚未系统地比较独立解决此类任务序列与从先前解决的任务中进行顺序迁移的效果。我们利用一个多任务基准测试填补了这一空白，在该基准测试中，每个基础实例在两种环境下被扩展为五个相关任务：部分时间窗扩展和交换-加性时间重新分配。我们在通用的惩罚分数目标下，使用流行的局部搜索方法 LNS、VNS 和 LKH-3，将标准的从零开始协议与迭代协议进行了比较，后者利用前一任务的最佳路径初始化每个任务。我们的实验结果表明，迭代协议在渐进松弛设置中始终表现优异，在交换-加性变化下通常具有竞争力，且在更困难的实例上改进效果更为显著。

## Abstract
In many real-world settings, problem instances that need to be solved are quite similar, and knowledge from previous optimization runs can potentially be utilized. We explore this for the Traveling Salesperson problem with time windows (TSPTW), which often arises in settings where the travel-time matrix is fixed but time-window constraints change across related tasks. Existing TSPTW studies, however, have not systematically compared solving such task sequences independently with sequential transfer from previously solved tasks. We address this gap using a multi-task benchmark in which each base instance is expanded into five related tasks under two environments: partial time-window expansion and swap-additive time reassignment. We compare a standard from-scratch protocol with an iterative protocol that initializes each task from the best tour of the previous task, using the popular local search approaches LNS, VNS, and LKH-3 under a common penalized-score objective. Our experimental results show that the iterative protocol is consistently superior in the progressive-relaxation setting and generally competitive under swap-additive changes, with improvements increasing on more difficult instances.

### 1. 核心问题与整体含义（研究动机和背景）
在现实世界的物流和调度中，优化问题往往不是孤立出现的，而是具有高度相似性的序列任务（例如：每日配送路线中，城市间的距离固定，但客户要求的送货时间窗会发生微调）。
*   **核心问题**：当面对一系列相似的 TSPTW 任务时，**“从零开始求解”**（Standard Protocol）与**“利用前序任务的最优解作为当前任务的起点”**（Iterative Protocol/Sequential Transfer）相比，哪种策略更有效？
*   **研究背景**：虽然迁移优化（Transfer Optimization）在其他领域有所应用，但在 TSPTW 领域，尚未有研究系统地比较顺序迁移对不同局部搜索算法性能的影响。

### 2. 论文提出的方法论
论文对比了两种求解协议，并应用于三种主流的局部搜索算法。
*   **核心思想**：通过“顺序迁移”（Sequential Transfer），将任务 $T_{k-1}$ 找到的最佳路径作为任务 $T_k$ 的初始解，利用结构相似性加速收敛并提高可行解的获取率。
*   **目标函数（惩罚分数）**：
    使用公式 $score(\pi) = c(\pi) + CV(\pi) \cdot L$。其中 $c(\pi)$ 是路径成本，$CV(\pi)$ 是违反时间窗的总延迟，$L$ 是一个巨大的常数。该设计确保了算法优先寻找可行解（$CV=0$），在不可行时则最小化延迟。
*   **关键算法细节**：
    *   **LNS (大邻域搜索)**：采用破坏-修复范式，通过移除和贪婪插入城市来探索邻域。
    *   **VNS (变邻域搜索)**：使用重定位（Relocate）和 2-opt 算子，结合抖动（Shaking）机制跳出局部最优。
    *   **LKH-3**：目前解决 TSP 及其变体最强大的启发式算法之一，论文对其进行了修改，使其在达到固定的函数评估次数（FE）时停止。

### 3. 实验设计
*   **数据集**：基于经典的 TSPTW 基准实例（Langevin 等人及 Dumas 等人的研究），规模涵盖 $n=20$ 到 $200$ 个城市。
*   **任务生成场景（Benchmark）**：
    1.  **部分时间窗扩展（Partial Expansion）**：每步随机选择 10%-15% 的城市放宽其时间窗。这是一种“渐进松弛”环境，前序解在逻辑上更容易保持可行。
    2.  **交换-加性重新分配（Swap-Additive）**：通过随机交换路径中的两个位置并围绕新的到达时间重新生成时间窗。这是一种“破坏性”环境，任务间的结构连续性较弱。
*   **对比方法**：在 LNS、VNS、LKH-3 三种算法上分别运行“从零开始”和“迭代初始化”两种协议。

*   **算力分配**：每个任务分配了 **100,000 次函数评估（FE）** 的固定预算。
*   **实验规模**：每个实例、每个任务、每个协议均独立运行 **30 次**，以确保统计意义。

### 5. 实验数量与充分性
*   **实验量**：涵盖了 7 种问题规模（20-200）、2 种变化环境、3 种核心算法、5 个连续任务序列。总计进行了数千次独立实验运行。
*   **充分性与公平性**：
    *   使用了非参数检验（Mann-Whitney U test）来评估统计显著性（$\alpha = 0.05$）。
    *   所有算法在相同的 FE 预算下运行，确保了比较的公平性。
    *   实验设计考虑了从简单到复杂的不同环境，评估非常全面。

### 6. 主要结论与发现
*   **渐进松弛场景（环境1）**：迭代协议表现出**压倒性优势**。随着问题规模 $n$ 的增大，迭代协议在可行解获取率（Success Rate）和最终路径质量上均显著优于从零求解。
*   **破坏性场景（环境2）**：迭代协议依然具有**竞争力**，但在某些情况下与从零求解差异不显著。这说明当任务间变化剧烈时，先前知识的引导作用会减弱，但通常不会产生负面影响。
*   **规模效应**：问题越难（城市越多、约束越紧），迭代协议带来的性能提升越明显。

### 7. 优点
*   **现实意义强**：模拟了物流行业中常见的动态约束变化场景。
*   **算法覆盖广**：不仅测试了基础的 LNS/VNS，还引入了业界标杆 LKH-3，增强了结论的说服力。
*   **统计严谨**：通过大量重复实验和显著性检验，排除了随机性干扰。

### 8. 不足与局限
*   **任务序列长度有限**：实验仅模拟了 5 个连续任务，更长序列下的长期演化（如是否会陷入局部最优陷阱）尚不明确。
*   **约束类型单一**：仅关注了时间窗的变化，未考虑旅行时间矩阵（路况变化）或城市坐标同时改变的情况。
*   **计算开销度量**：虽然使用了 FE 作为指标，但未详细讨论迭代初始化在实际墙钟时间（Wall-clock time）上的额外开销（尽管通常很小）。
