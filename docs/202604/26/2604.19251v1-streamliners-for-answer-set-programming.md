## 摘要
流线化约束（Streamliner constraints）通过排除部分解空间来减小组合问题的搜索空间。我们将 StreamLLM 方法（该方法利用大语言模型为约束程序设计生成流线化约束）适配到了回答集程序设计（ASP）中。给定一个 ASP 编码和少量小型训练实例，我们提示多个大语言模型（LLM）提出候选约束。那些导致语法错误、使可满足实例变得不可满足或在所有训练实例上性能下降的候选约束将被丢弃。幸存的流线化约束与原始编码一起进行评估，我们报告了虚拟最佳编码（VBE）的结果，该编码为每个实例在原始编码及其流线化变体中选择最快的一个。在三个 ASP 竞赛基准测试（伙伴单元问题、推箱子、汉诺塔）中，VBE 实现了比原始编码高出 4 至 5 倍的加速。不同的大语言模型产生了语义多样的约束，而非仅仅是语法上的变体，这表明该方法捕捉到了真实的问题结构。

## Abstract
Streamliner constraints reduce the search space of combinatorial problems by ruling out portions of the solution space. We adapt the StreamLLM approach, which uses Large Language Models (LLMs) to generate streamliners for Constraint Programming, to Answer Set Programming (ASP). Given an ASP encoding and a few small training instances, we prompt multiple LLMs to propose candidate constraints. Candidates that cause syntax errors, render satisfiable instances unsatisfiable, or degrade performance on all training instances are discarded. The surviving streamliners are evaluated together with the original encoding, and we report results for a virtual best encoding (VBE) that, for each instance, selects the fastest among the original encoding and its streamlined variants. On three ASP Competition benchmarks (Partner Units Problem, Sokoban, Towers of Hanoi), the VBE achieves speedups of up to 4--5x over the original encoding. Different LLMs produce semantically diverse constraints, not mere syntactic variations, indicating that the approach captures genuine problem structure.

*   **研究动机**：
    *   **流线化约束（Streamliners）**是一种有效的优化手段，通过排除部分解空间（如打破对称性或添加隐含约束）来加速搜索，但手动编写这些约束需要深厚的领域知识。

    *   **LLM 的潜力**：借鉴 LLM 在约束程序设计（CP）中的成功经验（StreamLLM），探索其是否能理解 ASP 的非单调语义并生成有效的辅助约束。

### 2. 核心方法论
该研究提出了一套全自动化的流水线，主要步骤如下：
*   **输入准备**：提供原始 ASP 编码和少量可在几秒内解决的小型可满足训练实例。

*   **候选筛选（过滤机制）**：
    *   **语法检查**：剔除导致 Clingo 报错的约束。
    *   **正确性验证**：剔除导致原本可满足的训练实例变为不可满足的约束。
    *   **性能评估**：剔除在所有训练实例上都导致运行时间变长的约束。
*   **约束选择与评估**：
    *   在 30 分钟的训练窗口内循环生成。
    *   基于**虚拟最佳编码（VBE）**准则，选择在训练集上表现最好的 3 个约束组合。
    *   最终在测试集上评估原始编码、单一流线化编码、组合编码以及 VBE（即在所有变体中取每实例的最快值）。

### 3. 实验设计
*   **基准测试集**：选取了 2011 年 ASP 竞赛中的三个经典组合问题：
    1.  **伙伴单元问题（Partner Units Problem, PUP）**：复杂的配置问题。
    2.  **推箱子（Sokoban）**：经典的规划问题。

*   **实验规模**：针对每个问题进行了两次独立的完整运行（Run 1 & Run 2），以观察 LLM 生成结果的变异性。

    *   **局限性**：实验仅覆盖了 3 个领域，虽然深入但广度尚可进一步扩展。

### 6. 主要结论与发现
*   **显著加速**：在 VBE 分析下，该方法相比原始编码实现了 **4 到 5 倍** 的加速。
*   **语义多样性**：LLM 生成的不仅是简单的语法变体，还包括复杂的对称性破除、隐含约束，甚至是引入辅助谓词的子程序（如汉诺塔中的 `stable` 谓词）。
*   **互补性**：不同的流线化约束在不同实例上表现各异，这证明了“组合编码”或“实例特定选择”的必要性。
*   **反直觉发现**：研究者尝试手动优化 LLM 生成的约束（如减少接地规模），结果发现手动优化后的版本往往比 LLM 生成的原始版本运行更慢，说明 LLM 捕捉到的某些模式更符合求解器的内部启发式机制。

### 7. 优点与亮点
*   **全自动化**：无需人工干预，无需预设复杂的语言偏置或领域模板。
*   **零样本/少样本学习**：仅需极少的小型实例即可引导 LLM 生成适用于大规模实例的有效约束。
*   **跨模型协作**：利用了当前主流 LLM 的集体智慧，提高了生成约束的质量和多样性。

### 8. 不足与局限
*   **VBE 的理想化**：VBE 是一种“先验预测”的上限，实际应用中如何为特定实例预先选择最合适的流线化变体仍是一个挑战（需要算法选择技术）。
*   **接地（Grounding）风险**：部分 LLM 生成的约束具有二次方级别的接地规模，在极大规模实例上可能导致内存溢出。
*   **正确性风险**：虽然通过了训练集验证，但流线化约束在理论上可能排除某些解，对于需要找到“所有解”的任务可能不适用。
*   **领域覆盖**：目前仅验证了决策类问题，尚未广泛测试优化类问题（尽管结论中提到这是未来方向）。
