## 摘要
奖励模型（RM）是将大语言模型（LLM）与人类偏好对齐的核心，为 RLHF 和高级解码策略提供动力。尽管先前的大多数工作集中在单步生成上，但现实世界的应用越来越多地采用多阶段 LLM 流水线，而有效的奖励引导在这些领域仍有待探索。我们通过组合优化的代码生成对此进行了研究，构建了一个将奖励模型集成到公式化和求解阶段的流水线。我们发现了一个关键挑战：奖励模型预测与实际流水线执行结果之间存在不一致性。为了解决这一问题，我们提出了流水线自适应奖励模型（PARM），该模型利用流水线特定数据和直接偏好优化来使奖励与下游反馈保持一致。我们将 PARM 实例化为一个两阶段流水线（公式化 -> 代码生成），并在四个公开优化基准测试上对其进行评估，对比基准模型和采样方法测量其执行率和求解准确率。在 GSM8K 上进行的补充跨领域实验评估了其迁移能力。结果表明，PARM 持续提升了流水线输出的质量和稳定性，为多阶段 LLM 推理的奖励建模提供了新的见解。

## Abstract
Reward models (RMs) are central to aligning large language models (LLMs) with human preferences, powering RLHF and advanced decoding strategies. While most prior work focuses on single-step generation, real-world applications increasingly adopt multi-stage LLM pipelines, where effective reward guidance remains underexplored. We investigate this through code generation for combinatorial optimization, constructing a pipeline that integrates reward models into both formulation and solution stages. We identify a critical challenge: inconsistency between reward model predictions and actual pipeline execution outcomes. To address this, we propose the Pipeline-Adapted Reward Model (PARM), which leverages pipeline-specific data and direct preference optimization to align rewards with downstream feedback. We instantiate PARM as a two-stage pipeline (formulation -> code generation) and evaluate it on four public optimization benchmarks, measuring execution rate and solving accuracy against baselines and sampling methods. A supplementary cross-domain experiment on GSM8K assesses transferability. Results demonstrate that PARM consistently improves pipeline output quality and stability, providing new insights into reward modeling for multi-stage LLM reasoning.

*   **研究动机**：随着 LLM 应用从简单的单步生成转向复杂的、多阶段的流水线（如：问题建模 -> 代码编写 -> 执行验证），传统的奖励模型通常只针对单个环节的局部质量进行评分，而忽略了该环节对下游任务的最终贡献。
*   **核心问题**：局部高分的中间输出（如一个看似正确但过于复杂的数学公式）可能导致下游环节失败（如代码运行超时或报错）。这种“局部优、全局劣”的现象限制了多阶段 LLM 系统的可靠性。

### 2. 方法论
*   **核心思想**：将奖励模型的训练目标从“评估局部偏好”转向“预测全局成功”。通过利用流水线最终的执行反馈（如代码是否运行成功、答案是否正确）来自动标注中间阶段的偏好数据。
*   **关键技术细节**：
    *   **两阶段架构**：以组合优化问题为例，分为“公式生成器（FG）”和“代码生成器（SG）”。
    *   **跨阶段奖励公式**：定义阶段奖励为最终验证结果的指示函数。即：如果某个中间公式 $F$ 最终能引导出正确的代码 $S$，则该公式被视为正样本。
    *   **自动化数据采集**：无需人工标注。通过对同一个问题采样多个候选方案并执行，根据执行结果（成功/失败）自动构建偏好对（Chosen/Rejected）。
    *   **训练算法**：采用 **DPO (Direct Preference Optimization)** 对奖励模型进行微调，使其学习到哪些中间特征更能导致下游的成功。
    *   **自调试集成**：在代码生成阶段引入错误反馈循环，利用奖励模型在调试过程中筛选最优的修正方案。

### 6. 主要结论与发现
*   **性能超越闭源模型**：PARM 驱动的轻量级专家模型流水线（如 Qwen2.5-7B 组合）在特定任务上的准确率超过了 GPT-4o 和 DeepSeek-v3。
*   **奖励模型的可扩展性**：随着候选样本数 $N$ 的增加，PARM 表现出持续的性能提升，而随机选择则会遇到瓶颈。
*   **跨领域有效性**：在 GSM8K 上的实验证明，PARM 的逻辑（计划 -> 求解）同样适用于数学推理，具有良好的通用性。

### 7. 优点
*   **零人工成本**：完全依赖执行反馈自动生成训练数据，解决了奖励模型训练数据昂贵且难以获取的问题。
*   **模块化强**：可以灵活更换流水线中的生成器或奖励模型，具有很强的工程实用性。
*   **解决信用分配问题**：通过跨阶段反馈，有效地将最终的成功归因于早期的正确决策。

### 8. 不足与局限
*   **阶段数限制**：目前主要验证了两阶段流水线。对于 3 阶段及以上的系统，由于“成功”信号在早期阶段会变得极其稀疏，训练难度和数据需求会呈指数级增长。
*   **依赖客观验证器**：该方法要求任务必须有明确的客观评价标准（如代码运行结果、数学唯一解）。对于创意写作、开放式对话等主观性强的任务，难以应用。
*   **计算开销**：Best-of-N 采样虽然效果好，但在推理时需要生成大量候选样本，计算成本较高。
