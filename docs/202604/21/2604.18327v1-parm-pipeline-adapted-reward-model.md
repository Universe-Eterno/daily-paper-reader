## 摘要
奖励模型（RM）是使大语言模型（LLM）与人类偏好对齐的核心，为基于人类反馈的强化学习（RLHF）和高级解码策略提供动力。尽管先前的大多数工作集中在单步生成上，但现实世界的应用越来越多地采用多阶段 LLM 流水线，而有效的奖励引导在这些场景中仍有待探索。我们通过组合优化的代码生成来研究这一问题，构建了一个将奖励模型整合到建模（formulation）和求解阶段的流水线。我们发现了一个关键挑战：奖励模型预测与实际流水线执行结果之间存在不一致性。为解决这一问题，我们提出了流水线适配奖励模型（PARM），该模型利用流水线特定数据和直接偏好优化来使奖励与下游反馈保持一致。我们将 PARM 实例化为一个两阶段流水线（建模 -> 代码生成），并在四个公开优化基准测试上对其进行评估，对比基准模型和采样方法，测量其执行率和求解准确率。在 GSM8K 上进行的补充跨领域实验评估了其迁移能力。结果表明，PARM 持续提升了流水线输出的质量和稳定性，为多阶段 LLM 推理的奖励建模提供了新的见解。

## Abstract
Reward models (RMs) are central to aligning large language models (LLMs) with human preferences, powering RLHF and advanced decoding strategies. While most prior work focuses on single-step generation, real-world applications increasingly adopt multi-stage LLM pipelines, where effective reward guidance remains underexplored. We investigate this through code generation for combinatorial optimization, constructing a pipeline that integrates reward models into both formulation and solution stages. We identify a critical challenge: inconsistency between reward model predictions and actual pipeline execution outcomes. To address this, we propose the Pipeline-Adapted Reward Model (PARM), which leverages pipeline-specific data and direct preference optimization to align rewards with downstream feedback. We instantiate PARM as a two-stage pipeline (formulation -> code generation) and evaluate it on four public optimization benchmarks, measuring execution rate and solving accuracy against baselines and sampling methods. A supplementary cross-domain experiment on GSM8K assesses transferability. Results demonstrate that PARM consistently improves pipeline output quality and stability, providing new insights into reward modeling for multi-stage LLM reasoning.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：传统的奖励模型主要针对单步文本生成进行优化。然而，在处理复杂任务（如组合优化问题）时，通常需要多阶段流水线（例如：自然语言描述 $\rightarrow$ 数学建模 $\rightarrow$ 代码生成 $\rightarrow$ 求解）。
*   **核心问题**：现有的奖励模型往往只关注局部阶段的质量，而忽略了局部输出对下游阶段的影响。例如，一个建模结果在数学上可能看起来很专业，但却过于复杂导致后续代码生成失败。这种“局部高分、全局失败”的现象限制了 LLM 流水线的可靠性。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：提出“流水线适配训练（PAT）”，利用流水线最终的执行反馈（如代码是否运行成功、答案是否正确）作为监督信号，来训练中间阶段的奖励模型。
*   **关键技术细节**：
    *   **两阶段架构**：将任务分解为“建模生成器（FG）”和“代码生成器（SG）”，每个阶段配有专门的奖励模型（$RM_F$ 和 $RM_S$）。
    *   **跨阶段效用函数**：定义奖励信号时，建模阶段的奖励取决于该模型是否能引导出至少一个正确的下游代码解；代码阶段的奖励则取决于代码是否通过验证。
    *   **自动化数据构建**：通过在训练集上运行流水线，自动收集“成功”和“失败”的样本对，无需人工标注。
    *   **DPO 优化**：采用直接偏好优化（Direct Preference Optimization, DPO）算法，使奖励模型学习到哪些中间输出更有利于最终任务的成功。
    *   **自调试循环**：在代码生成阶段引入错误反馈机制，利用奖励模型在调试过程中筛选最佳修正方案。

### 6. 主要结论与发现
*   **性能提升**：PARM 在所有数据集上均显著提升了执行率和准确率，甚至超过了参数量巨大的单体模型（如 GPT-4o）。
*   **适配的重要性**：经过流水线适配训练的奖励模型，其判别能力显著强于通用的奖励模型。
*   **非冗余贡献**：建模阶段和代码阶段的奖励模型提供了互补的信号，共同作用时效果最佳。
*   **可迁移性**：该方法在 GSM8K 上的成功证明了其不仅限于优化问题，也可扩展至其他多阶段推理任务。

### 7. 优点（亮点）
*   **无需人工标注**：完全依赖执行反馈自动生成偏好数据，极大地降低了训练成本。
*   **模块化与灵活性**：可以轻松更换流水线中的专家模型或奖励模型，具有很强的工程实用性。
*   **解决信用分配问题**：通过跨阶段反馈，有效地将最终的成功归因于中间的关键决策。

### 8. 不足与局限
*   **奖励稀疏性风险**：在超过 2 阶段的长流水线中，早期阶段获得正向反馈的概率会指数级下降，可能导致训练数据极度不平衡。
*   **领域局限**：虽然做了 GSM8K 实验，但核心验证仍集中在代码生成和数学优化领域，对于主观性较强（如创意写作流水线）的任务适用性待考证。
*   **计算开销**：Best-of-N 采样虽然效果好，但在推理时需要生成大量候选样本，增加了计算延迟和成本。
