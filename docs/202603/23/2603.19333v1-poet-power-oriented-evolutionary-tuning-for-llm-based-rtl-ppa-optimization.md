## 摘要
将大语言模型（LLM）应用于 RTL 代码优化以改进功耗、性能和面积（PPA）面临两个关键挑战：一是在 LLM 存在幻觉的情况下确保优化设计的功能正确性；二是在多目标 PPA 权衡空间中系统地优先考虑降低功耗。我们提出了 POET（Power-Oriented Evolutionary Tuning），这是一个应对上述挑战的框架。针对功能正确性，POET 引入了一种基于差分测试的测试平台（testbench）生成流水线，将原始设计视为功能参考模型（oracle），利用确定性仿真生成黄金参考（golden references），从而在验证过程中消除 LLM 幻觉。针对 PPA 优化，POET 采用了一种 LLM 驱动的进化机制，结合非支配排序、功耗优先的层内排名以及比例幸存者选择，在无需手动调整权重的情况下，引导搜索向 Pareto 前沿的低功耗区域靠拢。在包含 40 个不同 RTL 设计的 RTL-OPT 基准测试上进行的评估表明，POET 实现了 100% 的功能正确性，在所有 40 个设计中均获得了最佳功耗，并在面积和延迟改进方面具有竞争力。

## Abstract
Applying large language models (LLMs) to RTL code optimization for improved power, performance, and area (PPA) faces two key challenges: ensuring functional correctness of optimized designs despite LLM hallucination, and systematically prioritizing power reduction within the multi-objective PPA trade-off space. We propose POET (Power-Oriented Evolutionary Tuning), a framework that addresses both challenges. For functional correctness, POET introduces a differential-testing-based testbench generation pipeline that treats the original design as a functional oracle, using deterministic simulation to produce golden references and eliminating LLM hallucination from the verification process. For PPA optimization, POET employs an LLM-driven evolutionary mechanism with non-dominated sorting, power-first intra-level ranking, and proportional survivor selection to steer the search toward the low-power region of the Pareto front without manual weight tuning. Evaluated on the RTL-OPT benchmark across 40 diverse RTL designs, POET achieves 100% functional correctness, the best power on all 40 designs, and competitive area and delay improvements.

在集成电路设计中，将大语言模型（LLM）用于 RTL（寄存器传输级）代码优化面临两大挑战：
*   **功能正确性难题**：由于 LLM 固有的“幻觉”现象，生成的优化代码常包含功能错误。现有的验证方法（如手动编写测试平台或让 LLM 自行生成）要么效率低下，要么不可靠。
*   **多目标 PPA 权衡的复杂性**：功耗（Power）、性能（Performance/Delay）和面积（Area）之间存在复杂的权衡。现有方法通常采用加权求和（Weighted-sum）将多目标转为单目标，这需要繁琐的手动调参，且难以系统性地优先实现低功耗目标。

**POET 框架**旨在通过自动化验证流水线和面向功耗的进化算法，在保证 100% 功能等效的前提下，系统性地搜索 PPA 空间中的低功耗 Pareto 最优解。

### 2. 方法论
POET 包含两个核心组件：

*   **基于差异测试的测试平台生成（Differential-Testing-Based TB Generation）**：
    *   **核心思想**：将原始（虽然 PPA 非最优但功能正确）的 RTL 设计视为“功能参考模型（Oracle）”。
    *   **流程**：1. LLM 提取原始设计的规格说明；2. LLM 生成测试激励（输入向量）；3. 使用确定性仿真器运行原始设计，捕获“黄金输出”；4. 组装成自动检查的测试平台。
    *   **优势**：输出结果来自仿真而非 LLM 推理，彻底消除了验证环节的幻觉风险。

*   **面向功耗的进化优化引擎**：
    *   **进化算子**：定义了六种 LLM 驱动的算子（改进、重构、探索、简化、融合、交叉），并在提示词中嵌入了电路级优化知识（如门控时钟、操作数隔离等）。
    *   **自适应策略**：使用 UCB（置信上限）算法动态选择表现更好的算子。
    *   **幸存者选择机制**：
        1.  **非支配排序**：基于 Pareto 支配关系将种群分层。
        2.  **层内功耗排名**：在同一 Pareto 层内，按功耗升序排列，确保低功耗个体获得更高优先级。
        3.  **比例名额分配**：按优先级为各层分配幸存名额，既保留了低功耗偏好，又维持了种群多样性。
