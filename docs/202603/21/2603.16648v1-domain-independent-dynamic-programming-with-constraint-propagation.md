## 摘要
组合问题存在两种流行的基于模型的范式：1) 基于状态的表示，例如启发式搜索、动态规划 (DP) 和决策图；2) 基于约束和领域的表示，例如约束规划 (CP)、(混合) 整数规划和布尔可满足性。在本文中，我们通过将约束传播集成到 DP 中，弥合了 DP 和 CP 范式之间的差距，使 DP 求解器能够利用约束传播来剪枝状态和转移。为此，我们在领域无关动态规划框架中使用通用 CP 求解器实现了约束传播，并在三个组合优化问题上使用启发式搜索进行了评估：带时间窗的单机调度问题、资源受限项目调度问题 (RCPSP) 以及带时间窗的旅行商问题 (TSPTW)。我们的评估表明，约束传播显著减少了状态扩展的数量，使得我们的方法在单机调度和 RCPSP 问题上比 DP 求解器解决了更多的实例，并在高度受限的 TSPTW 实例中表现出类似的改进。运行性能表明，对于受限实例，传播带来的收益超过了开销，但进一步减少传播开销的研究可以进一步提高性能。我们的工作是理解 DP 求解器中约束传播价值的关键一步，为集成 DP 和 CP 提供了一种基于模型的方法。

## Abstract
There are two prevalent model-based paradigms for combinatorial problems: 1) state-based representations, such as heuristic search, dynamic programming (DP), and decision diagrams, and 2) constraint and domain-based representations, such as constraint programming (CP), (mixed-)integer programming, and Boolean satisfiability. In this paper, we bridge the gap between the DP and CP paradigms by integrating constraint propagation into DP, enabling a DP solver to prune states and transitions using constraint propagation. To this end, we implement constraint propagation using a general-purpose CP solver in the Domain-Independent Dynamic Programming framework and evaluate using heuristic search on three combinatorial optimisation problems: Single Machine Scheduling with Time Windows, the Resource Constrained Project Scheduling Problem (RCPSP), and the Travelling Salesperson Problem with Time Windows (TSPTW). Our evaluation shows that constraint propagation significantly reduces the number of state expansions, causing our approach to solve more instances than a DP solver for Single Machine Scheduling and RCPSP, and showing similar improvements for tightly constrained TSPTW instances. The runtime performance indicates that the benefits of propagation outweigh the overhead for constrained instances, but that further work into reducing propagation overhead could improve performance further. Our work is a key step in understanding the value of constraint propagation in DP solvers, providing a model-based approach to integrating DP and CP.

组合优化领域存在两大主流范式：**基于状态的表示**（如动态规划 DP、启发式搜索）和**基于约束的表示**（如约束规划 CP、整数规划 IP）。
*   **研究动机**：DP 擅长处理重复状态检测、优势检测（Dominance Detection）和启发式搜索，但在利用约束进行推理剪枝方面较弱；而 CP 擅长通过约束传播（Constraint Propagation）缩小变量范围，但在处理状态重叠和启发式引导方面不如 DP。
*   **核心问题**：如何将 CP 的通用约束传播技术集成到领域无关动态规划（DIDP）框架中，利用两者的互补优势来解决复杂的组合优化问题。

#### 2. 论文提出的方法论
论文提出了一种将约束传播集成到 DP 搜索过程中的通用框架，核心思想是为问题提供“双重视图”：
*   **核心思想**：在 DP 搜索的每个状态扩展阶段，调用一个通用的 CP 求解器。CP 求解器根据当前 DP 状态更新变量定义域并执行传播。
*   **关键技术细节**：
    *   **状态剪枝**：如果 CP 传播发现当前状态不可行（定义域为空），则直接剪枝。
    *   **对偶界限增强**：利用 CP 传播后的变量下界（LB）来强化 DP 的对偶界限（Dual Bound），从而提高 A* 或 CABS 搜索的效率。
    *   **转换过滤**：利用 CP 结果判断某个状态转换（Transition）是否会导致不可行，从而减少无效的后继状态生成。
*   **算法流程**：在生成后继状态的函数 `GenSucc` 中嵌入 `Propagate` 过程。首先根据当前状态 $S$ 构建/更新 CP 模型，执行传播得到缩小后的定义域 $D'$，若 $D'$ 导致不可行或界限超过当前最优解则停止扩展；否则，利用 $D'$ 过滤掉不满足约束的后继状态。

#### 6. 论文的主要结论与发现
*   **显著剪枝**：约束传播极大地减少了 DP 搜索过程中的状态扩展数量，在某些实例中减少了数个数量级。
*   **性能提升**：在单机调度和 RCPSP 问题上，集成 CP 的方法比纯 DP 解决了更多实例。
*   **约束敏感性**：该方法在**强约束（Tightly Constrained）**实例上表现极其出色。例如，在 TSPTW 中，当时间窗非常紧时，CABS+CP 远优于纯 CABS。
*   **开销权衡**：约束传播会带来额外的计算开销。在约束较松的实例中，传播带来的剪枝收益可能无法抵消其时间成本。

#### 7. 优点
*   **通用性**：提供了一种模型无关（Domain-Independent）的集成方式，不依赖于特定问题的启发式推理。
*   **范式融合**：成功将 CP 的强推理能力引入到 DP 的状态空间搜索中，为混合求解器设计提供了新思路。
*   **开源贡献**：提供了完整的代码和实验数据，便于社区复现和进一步研究。

#### 8. 不足与局限
*   **传播开销**：在每个状态都调用 CP 求解器会导致显著的运行负担，论文指出未来需要研究如何减少冗余计算或选择性地执行传播。
*   **松约束表现一般**：对于约束较少的问题，该方法可能不如纯 DP 快速，因为剪枝效果不明显。
*   **内存压力**：虽然减少了状态扩展，但 DP 框架本身在处理大规模状态空间时仍面临内存瓶颈。
