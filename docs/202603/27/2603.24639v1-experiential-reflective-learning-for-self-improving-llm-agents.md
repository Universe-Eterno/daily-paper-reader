## 摘要
大语言模型（LLMs）的最新进展使得开发能够进行复杂推理和多步问题解决的自主智能体成为可能。然而，这些智能体难以适应特定环境，且未能利用过去的交互经验，无论积累了多少经验，在面对新任务时往往都从零开始。我们提出了经验反思学习（Experiential Reflective Learning, ERL），这是一个简单的自我改进框架，通过经验学习实现快速的环境适应。ERL 通过对任务轨迹和结果进行反思来生成启发式策略，捕捉可跨任务迁移的可操作教训。在测试阶段，系统根据当前任务检索相关的启发式策略，并将其注入智能体的上下文以指导执行。在 Gaia2 基准测试中，ERL 的成功率比 ReAct 基线提高了 7.8%，在任务完成可靠性方面有显著提升，并优于先前的经验学习方法。通过系统的消融实验，我们发现选择性检索至关重要，且启发式策略比少样本轨迹提示提供了更具迁移性的抽象。这些结果表明，通过反思单次尝试的经验来提取可迁移的启发式策略，能够实现有效的智能体自我改进。

## Abstract
Recent advances in large language models (LLMs) have enabled the development of autonomous agents capable of complex reasoning and multi-step problem solving. However, these agents struggle to adapt to specialized environments and do not leverage past interactions, approaching each new task from scratch regardless of their accumulated experience. We introduce Experiential Reflective Learning (ERL), a simple self-improvement framework that enables rapid environment adaptation through experiential learning. ERL reflects on task trajectories and outcomes to generate heuristics, capturing actionable lessons that transfer across tasks. At test time, relevant heuristics are retrieved based on the current task and injected into the agent's context to guide execution. On the Gaia2 benchmark, ERL improves success rate by 7.8% over a ReAct baseline, with large gains in task completion reliability, and outperforms prior experiential learning methods. Through systematic ablations, we find that selective retrieval is essential and that heuristics provide more transferable abstractions than few-shot trajectory prompting. These results demonstrate that reflecting on single-attempt experiences to extract transferable heuristics enables effective agent self-improvement.

### 2. 方法论：核心思想与技术细节
ERL 框架包含两个核心阶段：
*   **启发式策略生成（Heuristic Generation）**：
    *   智能体在完成任务后，对执行轨迹（推理步骤、工具调用、输出）和结果（成功/失败信号）进行反思。
    *   生成结构化的**启发式策略（Heuristic）**，包含：1) **分析**（识别导致成功或失败的关键点）；2) **学习指南**（明确触发条件和推荐的操作步骤，例如：“当发送邮件给日历参与者时，先通过联系人工具解析姓名”）。
    *   这些策略存储在持久化的经验池中。
*   **检索增强执行（Retrieval-augmented Execution）**：
    *   面对新任务时，使用 LLM 作为评分器，根据任务描述的相似性、经验多样性和指南的信息量，从经验池中检索 Top-k（实验中 k=20）个相关的启发式策略。
    *   将检索到的策略注入智能体的系统提示词（System Prompt）中，指导其执行新任务。

### 6. 主要结论与发现
*   **性能提升**：ERL 在 Gaia2 上的总成功率达到 56.1%，比 ReAct 基线提升了 **7.8%**，且优于 ExpeL 和 AutoGuide。
*   **可靠性增强**：ERL 显著提升了 `pass^3`（连续三次成功的概率），表明学习到的启发式策略使智能体表现更稳定。
*   **抽象胜于原始**：经过反思提炼的“启发式策略”比原始的“任务轨迹”具有更好的迁移性，且在相同 Token 预算下效果更佳。
*   **经验类型差异**：**失败经验**对“搜索类”任务帮助最大（提供负面约束），而**成功经验**对“执行类”任务更有利（强化证明有效的序列）。

### 7. 优点（亮点）
*   **单次尝试学习**：与需要多次重试的方法不同，ERL 可以从单次任务轨迹中提取有效教训，更符合实际部署场景。
*   **高迁移性**：通过将具体经验抽象为“触发条件+动作”的指南，实现了跨任务的知识迁移。
*   **即插即用**：无需修改模型参数，通过 Prompt 工程即可实现智能体的自我改进。

### 8. 不足与局限
*   **检索开销**：虽然有缓存优化，但长上下文带来的 Token 消耗和推理延迟仍高于基础智能体。
*   **依赖奖励信号**：虽然实验证明在无外部奖励（靠智能体自评）时仍有一定效果，但准确的成功/失败反馈对生成高质量策略至关重要。
*   **复杂交互限制**：在 $\tau^2$-bench 的电信（Telecom）领域表现不佳，反映出该方法在处理高度随机的用户交互和组合式子任务时存在局限。
*   **冲突处理**：随着经验池扩大，如何处理相互冲突的启发式策略仍是一个待解决的挑战。
